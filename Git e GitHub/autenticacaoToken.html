Autenticação via TOKEN.

Anteriormente a autenticação do GITHUB era realizada através de login e senha. Atualmente (desde 2021) não é mais permitido.

Os token são exclusivos e também irrevogáveis. Com limitação (permissões) aos quais podemos definir. E evitável quanto ao uso de Brute Force.


Para criar um respositório novo basta acessar o site do GitHUB, clicando em mais, selecione a opção "Create a new repository". É importante adicionar um nome,caso queira, uma descrição, selecionar se é Público ou Privado, e inicializar o respositório com o README.


Para criar um Token devemos acessar o canto superior direito, clicar em settings, ir até o final o final da página e clicar em "Developer Settings".
Selecionar a opção "Personal acessess tokens", e depois "Tokens (Classic)".
Em seguida selecionar "Generate new token" e sem seguida, "Generate new token (Classic)".

Ao se criar um token, devemos selecionar o seu período e colocar uma anotação para o que é esse token. Depois devemos selecionar suas permissões.

Atenção! guarde esse token pois será apresentado somente uma única vez no site GitHUB no caso.

Como salvar o TOKEN?

Primeiro devemos usar o comando: "git config credential.helper store/cache" (Store é para deixar armazenado permanentemente e o cache fica temporariamente.)
