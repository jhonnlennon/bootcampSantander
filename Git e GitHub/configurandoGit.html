Lista de comandos para configurações.

  • git config
      • Permite visualizar e configurar variaveis obviamente de configuração.

  • git config --global user.name
      • O "git config" vai chamar a configuração;
      O "--global" indica que vai ficar armazenado para TODOS.
      O "user.name" indica que você vai definir o nome.

  • git config --global user.email
      • Agora você esta adicionando o e-mail.

  • git config user.name ou user.email
      • Com esse comando você vai estar visualizando o que esta configurado.

  • git config init.defaulBranch
      • Vai retornar no terminal o nome da sua branch padrão.
      Para alterar, utilize: git config --global init.defaultBranch nomeaqui

  • git config --global --list
      • Vai retornar todas as configurações feitas no global.

  • git config credential.helper store/cache
      • Usado para armazenar um TOKEN. (Store é para deixar armazenado permanentemente e o cache fica temporariamente). Para visualizar o que foi armazenado, utilize: git config --global credential.helper

  • git config --global --show-origin credential.helper
      • O "git config" vai chamar a configuração;
      O "--global" indica que vai ficar armazenado para TODOS;
      O "--show-origin" mostra o caminho de onde esta armazenado;
      O "credential.helper" variável de configuração.

      1. cat .gitconfig
          2. O "cat" vai mostrar o conteúdo do arquivo;
          O ".gitconfig" é o nome do arquivo, poderia ser qualquer outro..

      2. cat .git-credential
          3. O "cat" vai mostrar o conteúdo do arquivo;
          O ".git-credential" é o nome do arquivo, neste exemplo ele vai mostrar o token de acesso do gitHUB..

      3. ls -al ~/.ssh
          4. O "ls" vai listar o que esta na pasta;
          O "-al" indica listar TODOS os arquivos incluindo pastas OCULTAS. Que são as pastar com o "ponto" na frente.
          O "~/" diretório home do usuário. Onde ficam todos os arquivos.
          O ".ssh" é o diretório.
